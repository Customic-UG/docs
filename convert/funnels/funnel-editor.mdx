## Überblick

Der Funnel Editor ist das zentrale Werkzeug, um Funnels visuell aufzubauen. Sie definieren den Ablauf Schritt für Schritt: Welche Fragen werden gestellt, wie verzweigt der Funnel je nach Antwort, und was passiert am Ende?

{/* Bild: Gesamtansicht des Editors */}

---

## Steps

Ein Funnel besteht aus mehreren **Steps** – einzelnen Schritten, die ein Nutzer nacheinander durchläuft. Jeder Step kann eine oder mehrere Fragen enthalten und wird auf einer eigenen Seite angezeigt.

    <Frame>
      <img src="/images/convert/funnels/funnel-editor/steps.png" />
    </Frame>

- Neue Steps lassen sich per Klick hinzufügen
- Die Reihenfolge kann per Drag & Drop angepasst werden
- Jeder Step erhält einen internen Namen zur besseren Orientierung

---

## Blöcke

Innerhalb eines Steps können verschiedene **Blöcke** hinzugefügt werden. Blöcke sind die Bausteine eines Funnels.

    <Frame>
      <img src="/images/convert/funnels/funnel-editor/blocks.png" />
    </Frame>

### Häufig verwendete Blocktypen

Zu den meistgenutzten Blöcken gehören unter anderem:

| Block | Beschreibung |
|---|---|
| **Choice** | Eine Frage mit vordefinierten Antwortoptionen – Nutzer wählen genau eine |
| **Multi-Select** | Mehrere Antworten können gleichzeitig ausgewählt werden |
| **Basic Input** | Freies Textfeld für z. B. Name, E-Mail oder Wohnort |
| **Heading / Standard Text** | Statischer Inhaltsblock – z. B. für Überschriften, Erklärungen oder Hinweise an den Interessenten |
| **Basic Button** | Schaltfläche zum Absenden von Antworten oder Formulardaten – leitet den Nutzer zum nächsten Step weiter |

Weitere Blocktypen sind direkt im Editor verfügbar.

## Logik & Conditions

Jeder Block kann mit **Conditions** versehen werden, die steuern, was nach einer Interaktion passiert – z. B. welcher Step als nächstes angezeigt wird oder ob das Formular abgesendet wird.

{/* Bild: Logik-Einstellungen an einem Block */}

Für beide Varianten stehen dieselben Aktionen zur Verfügung:

| Aktion |
|---|
| Nächster Schritt |
| Vorheriger Schritt |
| Gehe zu Schritt … |
| Weiterleitung |
| Formular absenden |
| Formular abbrechen |

### Einfache Conditions

Mit einer einfachen Condition legen Sie fest, welche Aktion bei einer Interaktion ausgeführt wird.

### Erweiterte Conditions

Mit erweiterten Conditions lassen sich Regeln mit beliebig vielen Bedingungen definieren. Es können mehrere **Rules** erstellt werden, und jede Rule kann mehrere Bedingungen enthalten.

    <Frame>
      <img src="/images/convert/funnels/funnel-editor/conditions.png" />
    </Frame>

Aufbau einer Bedingung:

1. **Wenn** – Auswahl des Blocks, dessen Wert geprüft wird
2. **Operator** – Gleich / Ungleich / Ist eins von / Ist gesetzt
3. **Wert** – der zu prüfende Wert
4. **Dann** – die Aktion, die ausgeführt wird

## Häufige Fragen

<AccordionGroup>
  <Accordion title="Kann ich einen Step nachträglich verschieben oder löschen?">
    Ja, Steps lassen sich jederzeit per Drag & Drop umsortieren oder über das Kontextmenü löschen. Verknüpfte Logik wird dabei entsprechend angepasst.
  </Accordion>
  <Accordion title="Was passiert, wenn ich keine Logik an einen Block setze?">
    Ohne explizite Logik führt der Funnel automatisch zum nächsten Step in der Reihenfolge weiter.
  </Accordion>
  <Accordion title="Kann ich einen Funnel nach der Veröffentlichung noch bearbeiten?">
    Ja, Änderungen können jederzeit vorgenommen werden. Nach dem Speichern sind sie sofort im Live-Funnel aktiv.
  </Accordion>
</AccordionGroup>
