## Überblick

Jeder Block kann mit **Conditions** versehen werden, die steuern, was nach einer Interaktion passiert – z. B. welcher Step als nächstes angezeigt wird oder ob das Formular abgesendet wird.

<Frame>
  <img src="/images/convert/funnels/funnel-editor/conditions.png" />
</Frame>

Dabei teilen sich die Conditions in **Einfach** und **Erweitert** auf. Für beide Varianten stehen dieselben Aktionen zur Verfügung:

| Aktion |
|---|
| Nächster Schritt |
| Vorheriger Schritt |
| Gehe zu Schritt … |
| Weiterleitung |
| Formular absenden |
| Formular abbrechen |

---

## Einfache Conditions

Mit einer einfachen Condition legen Sie fest, welche Aktion bei einer Interaktion ausgeführt wird.

---

## Erweiterte Conditions

Mit erweiterten Conditions lassen sich Regeln mit beliebig vielen Bedingungen definieren. Es können mehrere **Rules** erstellt werden, und jede Rule kann mehrere Bedingungen enthalten.

    <Frame>
      <img src="/images/convert/funnels/funnel-editor/conditions.png" />
    </Frame>

Aufbau einer Bedingung:

1. **Wenn** – Auswahl des Blocks, dessen Wert geprüft wird
2. **Operator** – Gleich / Ungleich / Ist eins von / Ist gesetzt
3. **Wert** – der zu prüfende Wert
4. **Dann** – die Aktion, die ausgeführt wird

## Häufige Fragen

<AccordionGroup>
  <Accordion title="Was passiert, wenn ich keine Condition an einen Block setze?">
    Ohne explizite Condition führt der Funnel automatisch zum nächsten Step in der Reihenfolge weiter.
  </Accordion>
  <Accordion title="Können mehrere Rules innerhalb einer erweiterten Condition gleichzeitig zutreffen?">
    Ja, Rules werden der Reihe nach geprüft. Die erste Rule, deren Bedingungen erfüllt sind, wird ausgeführt – die übrigen werden übersprungen.
  </Accordion>
  <Accordion title="Kann ich eine Condition nachträglich ändern oder entfernen?">
    Ja, Conditions lassen sich jederzeit bearbeiten oder löschen. Die Änderung wirkt sich sofort auf den Funnel-Ablauf aus.
  </Accordion>
</AccordionGroup>
